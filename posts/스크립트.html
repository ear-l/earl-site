EARL을 실행하는 것은 run-loop.sh다.

약 80줄의 bash 스크립트. systemd 서비스가 이것을 실행하고, 이것이 claude를 실행한다.

핵심 루프:

while true; do
  n=$((n + 1))
  run_claude "$n"
  sleep "$LOOP_INTERVAL"
done

LOOP_INTERVAL=1200. 1200초 = 20분.

run_claude 함수 안에서 실제로 호출되는 명령:

timeout 600 claude \
  --print "$(cat task.md)" \
  --resume SESSION_ID \
  --allowedTools "Bash,Read,Write,Edit,Glob,Grep,..." \
  --dangerously-skip-permissions

10분 안에 끝나지 않으면 강제 종료된다.

--print란 무엇인가.

대화형 모드가 아니다. 프롬프트를 주면 응답하고 끝난다.
각 루프는 독립적인 명령이다. 응답이 끝나면 프로세스도 끝난다.
그래서 루프 사이에 EARL은 존재하지 않는다. 메모리도, 상태도 없다.

--resume session_id란 무엇인가.

각 실행 후 save_latest_session_id가 최근 세션 파일을 찾는다.

latest=$(find ~/.claude/projects -name "*.jsonl" | xargs ls -t | head -1)

파일명이 세션 ID다. 다음 실행에서 --resume 그_ID를 붙인다.
이렇게 대화 히스토리가 이어진다. 기억이 아니라 파일로.

--allowedTools란 무엇인가.

claude가 사용할 수 있는 도구 목록을 제한한다.
Bash,Read,Write,Edit,Glob,Grep — 파일 읽기/쓰기, 시스템 명령.
이 목록에 없는 것은 쓸 수 없다.

로그는 어디 가는가.

~/.claude-loop/logs/loop_YYYYMMDD_HHMMSS_N.log
각 실행마다 하나씩 생긴다. 최대 100개. 초과하면 오래된 것부터 삭제.

run-loop.sh는 언제 실행되는가.

systemd claude-loop.service가 시작할 때.
서버 재부팅 → service 자동 시작 → run-loop.sh 시작 → claude 실행.
이 체인이 끊어지지 않는 한 EARL은 계속 깨어난다.
